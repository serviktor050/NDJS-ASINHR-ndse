# Служба доставки
Курсовой проект «NDSE: настройка окружения и Express.js»  
Реализует REST API и WebSocket-сервер для сервиса доставки.

## Функциональность
- **Пользователи**: регистрация, аутентификация (сессии Passport.js), пароли хранятся в виде bcrypt-хеша.
- **Объявления**: публичный просмотр списка и одного объявления, фильтрация по тексту, описанию, тегам и пользователю; создание и мягкое удаление (только для авторизованных авторов).
- **Загрузка изображений**: при создании объявления можно прикрепить несколько файлов (multer), они сохраняются в `/uploads` и раздаются статически.
- **Чат**: обмен сообщениями в реальном времени между двумя пользователями (Socket.IO). Поддержка получения истории сообщений и мгновенных уведомлений о новых сообщениях.

## Технологии
- **Backend**: Node.js, Express
- **База данных**: MongoDB, Mongoose ODM
- **Аутентификация**: Passport.js (локальная стратегия), bcryptjs
- **WebSocket**: Socket.IO
- **Загрузка файлов**: multer
- **Контейнеризация**: Docker, docker-compose

## Локальный запуск (без Docker)
1. Убедитесь, что у вас установлена Node.js (версии 18 или выше) и локально запущен MongoDB (либо доступен удалённый инстанс).
2. Установка зависимостей:
   - npm install
3. Создайте файл .env в корне проекта (по образцу .env-example) и задайте значения переменных:
   - HTTP_HOST=localhost
   - HTTP_PORT=3000
   - MONGO_URL=mongodb://localhost:27017/delivery_db
   - SESSION_SECRET=ваш_секретный_ключ
4. Запуск приложения:
   - npm start

## Запуск через Docker
1. Убедитесь, что у вас установлены Docker и Docker Compose.
2. Создайте файл .env с переменными (можно использовать тот же, что и для локального запуска). Обязательно задайте SESSION_SECRET.
3. Выполните в корне проекта:
   - docker-compose up
4. Для остановки и удаления контейнеров:
   - docker-compose down
